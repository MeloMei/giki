# giki-core Implementation Plan (Plan 1 of 2)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. Every task follows TDD: write failing tests → verify failure → implement → verify pass → commit.

**Goal:** Build the giki v0.1 core: `giki init`, `giki ingest`, `giki config` commands with a 3-step LLM compilation pipeline (Analyze → Synthesize → Crosslink), git-native commits, Obsidian-compatible output, and PDF/markdown/text sources.

**Architecture:** Pure CLI, no daemons, no IPC. State lives in the git repo + working tree. Typer for CLI, GitPython for git, pypdf for PDFs, httpx for LLM APIs (no litellm/langchain). Clean layering: `sources/` (input) → `orchestrator` (pipeline) → `llm/` (adapters) → `wiki/` (storage) → git commit.

**Tech Stack:** Python 3.11+, Typer, GitPython, pypdf, httpx, PyYAML, jsonschema, pytest, respx (HTTP mocking), vcrpy (LLM cassettes).

**Related spec:** `docs/superpowers/specs/2026-06-30-giki-v0.1-design.md` — **read the spec first**; this plan intentionally references spec sections rather than duplicating requirements.

**Sibling plan (after this one):** `giki-review` (Plan 2 of 2) — `giki review` command and GitHub Action template. **DO NOT implement `giki review` in this plan** — leave a `raise NotImplementedError` stub.
